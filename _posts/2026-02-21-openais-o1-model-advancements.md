## 💻 דוגמאות קוד מעשיות: כיצד o1 פותר בעיות אמיתיות

אחד היתרונות הגדולים של **o1** הוא היכולת לייצר קוד איכותי ויעיל. בואו נראה כמה דוגמאות קונקרטיות שבהן ניתן להשתמש ב-API של OpenAI כדי לנצל את היכולות שלו.

### דוגמה 1: פתרון בעיית מתמטיקה מורכבת
נניח שאתם צריכים לפתור משוואה דיפרנציאלית מסדר שני. **o1** ינתח את הבעיה צעד אחר צעד.

```python
import openai

client = openai.OpenAI(api_key="your-api-key")

response = client.chat.completions.create(
    model="o1-preview",
    messages=[
        {"role": "user", "content": "פתור את משוואת דיפרנציאלית: y'' + 4y' + 4y = e^{-2x}, עם תנאי התחלה y(0)=1, y'(0)=0. הסבר צעד אחר צעד."}
    ]
)

print(response.choices[0].message.content)
```

התשובה תכלול פתרון מדויק: \( y(x) = (1 + x)e^{-2x} \), עם הסבר מפורט על שיטת הפחתת סדר ומקדמי וריאציה.

### דוגמה 2: כתיבת אלגוריתם מתקדם לקודינג
בואו ניצור פונקציה שמיישמת **LeetCode-style** בעיה: מציאת המסלול הארוך ביותר בגרף ללא מחזורים.

```python
# בקשת API לדגם o1
response = client.chat.completions.create(
    model="o1-mini",
    messages=[
        {"role": "user", "content": """
        כתוב פונקציה ב-Python שמוצאת את המסלול הארוך ביותר בגרף מכוון ללא מחזורים (DAG).
        כניסה: רשימת קשתות graph = [[1,2],[2,3],[3,4]]
        פלט: אורך המסלול הארוך ביותר.
        השתמש ב-DP ותסביר את הלוגיקה.
        """}
    ]
)
```

**o1** ייצר קוד נקי:

```python
def longest_path(graph):
    n = max(max(edge) for edge in graph) + 1 if graph else 0
    dp = [1] * n
    for u in range(n):
        for v in graph[u]:
            dp[v] = max(dp[v], dp[u] + 1)
    return max(dp) - 1
```

> **טיפ פרקטי:** השתמשו ב-**o1-mini** למשימות קוד פשוטות יותר כדי לחסוך בעלויות – הוא עדיין מצוין!

### דוגמה 4: אופטימיזציה של קוד קיים
אם יש לכם קוד לא יעיל, **o1** יעשה refactoring אוטומטי.

```python
# קוד מקורי לא יעיל
def fibonacci(n):
    if n <= 1:
        return n
    return fibonacci(n-1) + fibonacci(n-2)

# בקשה ל-o1: "אופטימיזציה של הפונקציה הזו עם memoization."
```

תוצאה: שימוש ב-**lru_cache** להאצה דרמטית.
